*### After Variable Comments

  • After variable comments should really only be used in classes/structs/enums (basically something with a namespace)

  • Several different styles: ```cpp int x; //!< Brief description

    int y; ///< Brief description

    int z; /*! Detailed description */

    int a; /**< Detailed descripton */

    int b; //!< Detailed description //!< when spanning two lines in c; /// Another detailed description /// spanning two lines ```

  • After variable comments can also be used within a function signature (though it looks really ugly and obfuscates the code, in my opinion): ```cpp int sum(int x /**< [in] first integer */, int y /**< [in] second integer */) { ... } ```
